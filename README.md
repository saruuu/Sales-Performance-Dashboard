Sales Performance Dashboard & SQL Analysis
📊 Project Overview
This project provides a comprehensive analysis of sales data from 2003 to 2005. It combines PostgreSQL-driven data exploration with an Excel-based dashboard to identify key performance indicators (KPIs), seasonal trends, and high-value customer segments.

🎯 Business Objectives
Revenue Growth: Identifying top-performing product lines and regional sales drivers.

Customer Insights: Segmenting customers based on purchasing behavior and lifetime value.

Market Breakdown: Analyzing revenue distribution across different countries and deal sizes.

🛠️ Tech Stack
Database: PostgreSQL (Data cleaning and complex analytical queries).

Visualization: Excel (Pivot Tables and Interactive Sales Dashboard).

Skills: CTEs, Window Functions, Aggregate Functions, and Data Modeling.

📂 Project Structure
sales_analysis_queries.sql: The primary SQL script containing all analysis queries.

Sales_Dashboard.xlsx: The Excel workbook containing the final report and charts.

🚀 Key Insights from SQL Analysis
Top Product Line: Classic Cars leads the revenue share with $3.9M, approximately 39% of total revenue.

Seasonal Trends: November consistently shows a massive peak in revenue (Q4 spike) across all years.

Regional Leaders: The USA is the top-performing country with 36.2% of revenue.

Customer Concentration: The top 10 customers contribute over 15% of total revenue.

📝 How to Use
Clone the repository: git clone https://github.com/saruuu/Sales-Performance-Dashboard.git

Create the sales table in PostgreSQL using the schema provided in the SQL file.

Import the CSV data and execute the queries to reproduce the analysis.
